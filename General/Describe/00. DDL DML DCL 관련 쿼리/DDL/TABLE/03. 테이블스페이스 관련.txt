*********************** [ 테이블스페이스 개념(오라클 기준) ] ***********************
오라클 데이터베이스에서 테이블스페이스(TableSpace)는 모든 데이터 저장의 기본 단위이다.
데이터 파일(Data File)과 함께 스토리지 관리를 담당하며, 성능 최적화, 데이터 분리, 백업 전략을
수립하는 데 있어 중요한 역할을 한다.


*********************** [ 테이블스페이스의 구조(트리형태) ] ***********************
설명: 테이블스페이스는 다음과 같은 계층적 구조를 가진다.
 
오라클 데이터베이스
|
|--- 테이블스페이스(TABLESPACE)
|	 |
|	 |--- 데이터파일(Datafile)
|	 |	 |
|	 |	 |--- 세그먼트(Segment)			/* 가장 큰 단위			*/
|	 |	 |	  |> 여러개의 익스텐트로 구성됨
|	 |	 |	  |> 테이블 세그먼트			/* 테이블 데이터 저장		*/
|	 |	 |	  |> 인덱스 세그먼트			/* 인덱스 저장				*/
|	 |	 |	  |> 롤백 세그먼트				/* 트랜잭션 롤백 정보 저장	*/
|	 |	 |	  |> 임시 세그먼트				/* 정렬 및 임시 데이터 저장	*/
|	 |	 |	  |> 클러스터 세그먼트			/* 클러스터된 테이블 저장		*/
|	 |	 |	  |
|	 |	 |	  |--- 익스텐트(Extent)		/* 세그먼트를 구성하는 묶음	*/
|	 |	 |	  |	   |> 여러 개의 블록으로 구성됨
|	 |	 |	  |	   |
|	 |	 |	  |	   |--- 블록(Block)		/* 데이터 저장의 최소 단위 	*/
|	 |	 |	  |	   |	|> 헤더			/* 블록 정보 저장			*/
|	 |	 |	  |	   |	|> 데이터 영역		/* 실제 행(row) 데이터 저장	*/ 
|	 |	 |	  |	   |	|> 트랜잭션 정보	/* 트랜잭션 변경 사항 기록	*/
|	 |	 |	  |	   |	|> 프리 리스트		/* 빈 공간				*/ 
|	 |	 |	  |	   |	|> ITL			/* 트랜잭션 관리			*/
|	 |	 |	  |	   |	|


*********************** [ 테이블스페이스 개념정리 ] ***********************
	계층						단위					설명
테이블스페이스(Tablespace)		논리적 저장 공간		데이터 파일들의 논리적 그룹
데이터 파일(Datafile)			물리적파일				테이블스페이스를 구성하는 실제 파일
세그먼트(Segment)				논리적 객체			테이블, 인덱스 등 데이터 저장 공간
익스텐트(Extent)				블록 묶음				여러 개의 연속된 블록을 묶은 저장 공간
블록(Block)					최소 단위				데이터를 저장하는 가장 작은 단위
